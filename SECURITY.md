# Security Policy
We take the security of our users and their data very seriously.  
This document explains how to responsibly report potential vulnerabilities in our project.

---

## Reporting a Vulnerability

If you believe you’ve discovered a security vulnerability in this project, we appreciate your help in disclosing it responsibly.

### Please do **not**:
- Open a public issue.
- Share details of the vulnerability publicly.

### Instead, please:
- Email **security@kommenszlapf.website**, or  
- Use the **“Report a vulnerability”** link on our GitHub repository’s *Security* tab.

Include as much detail as possible:
- A clear description of the issue.
- Steps to reproduce it.
- The environment or version affected.
- Any possible mitigations or fixes you suggest.

---

## Responsible Disclosure

We ask that you:
- Give us reasonable time to investigate and fix the issue before publicizing it.
- Avoid testing vulnerabilities on production servers that could affect real users.
- Follow ethical security testing practices.

Thank you for helping keep our users and platform safe.
